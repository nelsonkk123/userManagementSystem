## 0. 文档说明

### 0.1 文档目标

明确后台管理系统在用户、模板、任务、数据、商业化、系统配置等方面的完整需求，为前端（Next.js）、后端（Supabase）与 AI 编程协作提供统一语义基础。

- 单次只读取一个模块
- 输出内容需严格基于当前章节，不得引入外部假设

用途声明：本章节用于定义文档边界与 AI 使用规范。

## 1. 项目概述

### 1.1 产品目标

- 支撑思拓智媒小程序的运营与商业化
- 覆盖用户管理、模板生态、视频任务、数据分析、权限与配置
- 支持 MVP 及后续版本持续扩展

### 1.2 后台核心角色

- 超级管理员
- 运营管理员
- 内容管理员
- 数据分析师

用途声明：本章节用于明确后台系统的业务目标与使用人群。

---

## 2. 技术与架构约束

### 2.1 前端技术栈

- Next.js App Router
- Tailwind CSS
- shadcn/ui + Radix
- Lucide Icons

### 2.2 后端与数据

- Supabase（PostgreSQL + Auth + Storage）
- Row Level Security（RLS）

用途声明：本章节用于限定技术实现边界，避免 AI 输出偏离技术栈。
